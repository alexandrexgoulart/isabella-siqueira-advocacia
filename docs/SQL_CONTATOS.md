# SQL - Criar Tabela de Contatos no Supabase (Versão Segura)

Execute o seguintes comandos no SQL Editor do Supabase. Cada comando pode ser executado separadamente.

---

##Passo 1: Verificar se a tabela já existe, se não, criar

```sql
-- Criar tabela de contatos (se não existir)
CREATE TABLE IF NOT EXISTS contatos (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nome TEXT NOT NULL,
    email TEXT NOT NULL,
    telefone TEXT,
    mensagem TEXT,
    lido BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

---

## Passo 2: Habilitar RLS (se não habilitado)

```sql
ALTER TABLE contatos ENABLE ROW LEVEL SECURITY;
```

---

## Passo 3: Criar políticas (cada uma separadamente)

```sql
-- Política de INSERT pública (para o site salvar contatos)
CREATE POLICY "public_insert_contatos" ON contatos
    FOR INSERT TO anon
    WITH CHECK (true)
    ON CONFLICT (policy_name_oid) DO NOTHING;
```

```sql
-- Política de SELECT para authenticated (admin ver contatos)
CREATE POLICY "auth_read_contatos" ON contatos
    FOR SELECT TO authenticated
    USING (true)
    ON CONFLICT (policy_name_oid) DO NOTHING;
```

```sql
-- Política de UPDATE para marcar como lido
CREATE POLICY "auth_update_contatos" ON contatos
    FOR UPDATE TO authenticated
    USING (true)
    WITH CHECK (true)
    ON CONFLICT (policy_name_oid) DO NOTHING;
```

```sql
-- Política de DELETE
CREATE POLICY "auth_delete_contatos" ON contatos
    FOR DELETE TO authenticated
    USING (true)
    ON CONFLICT (policy_name_oid) DO NOTHING;
```

---

## Passo 4: Grant permissions

```sql
-- Conceder permissões
GRANT SELECT ON public.contatos TO anon;
GRANT SELECT ON public.contatos TO authenticated;
GRANT INSERT ON public.contatos TO anon;
GRANT UPDATE ON public.contatos TO authenticated;
GRANT DELETE ON public.contatos TO authenticated;
```

---

## Verificar se foi criado com sucesso:

```sql
SELECT * FROM contatos LIMIT 10;
```

Se não der erro de tabela inexistente, está tudo pronto!

---

## Se der erro de "policy already exists":

Execute apenas o GRANT:

```sql
GRANT SELECT ON public.contatos TO anon;
GRANT SELECT ON public.contatos TO authenticated;
GRANT INSERT ON public.contatos TO anon;
GRANT UPDATE ON public.contatos TO authenticated;
GRANT DELETE ON public.contatos TO authenticated;
```
